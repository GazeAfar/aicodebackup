别让 AI 生成的代码丢失

AICodeBackup 是一个 AI 代码备份 CLI，把本地项目保存为私有 GitHub 恢复点。 在崩溃、重置或长时间 AI 编程会话结束前，先给重要工作留下一份安全副本。

npm install -g aicodebackup@alpha
安装 alpha 查看 GitHub
  • 默认私有
  • 支持 Windows 和 macOS

AI 写代码很快丢代码也很快

工作丢失

崩溃、重置或误删都可能让项目消失。

没有备份

AI 工具不会自动替你保存源码。

副本混乱

散落的文件夹和版本会让恢复变困难。

隐私风险

公开仓库或普通同步盘可能暴露敏感代码。

四步建立安全备份

第一次连接 GitHub，之后在终端里重复备份。

安装安全层

AICodeBackup 检查 Git 和 GitHub CLI，并在可行时安装缺失工具。

连接 GitHub

新用户可以创建账号，已有用户通过默认浏览器完成授权。

安全备份

项目默认备份到私有 GitHub 仓库，为每个有价值阶段留下恢复点。

持续监控

使用 watch 检测长会话、较大改动和过久未备份的风险。

覆盖完整流程的核心命令

初始化、备份、恢复、健康检查和风险监控都覆盖 AI 编程项目的关键保护流程。

aicodebackup setup

初始化项目，安装或检查依赖，连接 GitHub，并创建私有仓库。

aicodebackup backup

保存当前项目状态，提交并推送到 GitHub，形成安全恢复点。

aicodebackup doctor

检查 Git、GitHub CLI、登录状态、远程仓库和最近备份状态。

aicodebackup watch

监控文件变更、Diff 大小和备份间隔；也可使用 --auto 自动备份。

aicodebackup restore

列出最近远程备份，或恢复到当前项目外的新目录，避免覆盖本地工作。

常见问题

我需要会 Git 吗？

不需要。AICodeBackup 把常见 Git 设置、提交和推送包装成简单命令。

我需要 GitHub 账号吗？

需要。如果还没有账号，setup 会打开 GitHub 注册；登录后通过 GitHub CLI 授权。

我的代码真的是私有的吗？

是。创建新仓库时默认使用 GitHub 私有仓库。

代码保存在哪里？

代码保存在你自己的私有 GitHub 仓库中，而不是 AICodeBackup 托管的代码存储服务。

什么时候应该备份？

功能跑通后、危险跨文件 AI 改动前，以及长时间编码会话结束前都应该备份。

卸载后会怎样？

你的 GitHub 仓库和备份仍然保留在你的 GitHub 账号中。

AICodeBackup 免费吗？

alpha CLI 免费，并以 MIT License 开源。
