备份 Claude Code 项目。

Claude Code 适合在终端里做大范围项目编辑。越是能一次处理很多文件，越需要稳定的备份节奏。

快速答案

备份 Claude Code 项目的核心，是在大提示词前后建立恢复点。AICodeBackup 可以把当前目录连接到私有 GitHub 仓库，并用 doctor 检查远端状态。

这能减少长会话结束后才发现本地版本混乱、重要文件没保存或重构路径无法回退的情况。

试用备份流程

先用测试项目验证。安装 alpha 版本，在项目目录中运行 setup，再创建私有恢复点。

npm install -g aicodebackup@alpha
安装 alpha

Claude Code 会话为什么需要恢复点

终端驱动的 AI 编辑经常跨越源码、测试、配置和文档。没有恢复点时，回到上一个可运行状态会变得很困难。

建议流程

  1. 会话开始前运行 doctor。
  2. 输入大范围修改请求前运行 backup。
  3. 测试通过后再次运行 backup。
  4. 长会话中使用 watch 观察备份年龄。
  5. 恢复到新目录后再比较差异。

和手动 Git 的关系

如果你已经熟悉 Git，可以继续使用自己的分支和提交习惯。AICodeBackup 更适合需要简单安全层、或还没形成固定 Git 流程的用户。

FAQ

我需要先学会 Git 吗？

不需要。AICodeBackup 仍然使用 Git 和 GitHub，但日常流程是 setup、backup、watch、doctor 和 restore。

备份会让仓库公开吗？

不会。为项目创建新仓库时，setup 默认创建私有 GitHub 仓库。

这能替代专业 Git 工作流吗？

不能。它是 AI 编程用户的安全层，用来在完整 Git 习惯建立之前先获得可靠的私有恢复点。
