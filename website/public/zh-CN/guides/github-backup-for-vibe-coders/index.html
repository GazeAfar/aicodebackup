面向 vibe coders 的 GitHub 备份。

Vibe coding 的重点是快速试想法，但项目一旦只存在本地目录里，任何误删、重置或失败实验都会变成风险。

快速答案

GitHub 备份可以把 vibe coding 项目从单机文件夹变成私有远端恢复点。AICodeBackup 把常见 Git 操作包装成 setup、backup、doctor、watch 和 restore。

你仍然拥有自己的 GitHub 仓库和代码。这个工具只是帮你把第一条安全路径搭起来，减少因为不熟悉 Git 而迟迟不备份的情况。

试用备份流程

先用测试项目验证。安装 alpha 版本，在项目目录中运行 setup，再创建私有恢复点。

npm install -g aicodebackup@alpha
安装 alpha

为什么 vibe coding 更需要备份

AI 可以快速生成可运行原型，也可以快速把一个可运行版本改坏。你可能还没来得及整理分支、提交信息或文档，项目就已经跨过了几个重要阶段。

适合非传统开发者的流程

  1. 先在测试项目中安装并运行 setup。
  2. 确认 GitHub 私有仓库已经创建。
  3. 每次完成一个可用版本后运行 backup。
  4. 用 doctor 看当前目录是否已经推送到远端。
  5. 需要新副本时恢复到当前项目外的新文件夹。

隐私默认值

AICodeBackup 创建新仓库时默认使用私有仓库。这样更适合还在实验阶段的 AI 代码、配置文件和业务逻辑，避免未准备好时公开。

FAQ

我需要先学会 Git 吗？

不需要。AICodeBackup 仍然使用 Git 和 GitHub，但日常流程是 setup、backup、watch、doctor 和 restore。

备份会让仓库公开吗？

不会。为项目创建新仓库时，setup 默认创建私有 GitHub 仓库。

这能替代专业 Git 工作流吗？

不能。它是 AI 编程用户的安全层，用来在完整 Git 习惯建立之前先获得可靠的私有恢复点。
