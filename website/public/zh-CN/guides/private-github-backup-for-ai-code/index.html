AI 代码的私有 GitHub 备份。

AI 生成代码经常包含还未清理的业务逻辑、配置、实验代码和依赖选择。默认私有备份可以先保护工作，再决定是否公开。

快速答案

私有 GitHub 备份的目标不是隐藏开源项目，而是在项目准备好之前避免提前暴露。AICodeBackup 创建新仓库时默认使用私有仓库。

代码仍然保存在你的 GitHub 账号下，AICodeBackup 不提供托管代码存储服务，也不会接收你的源码内容。

试用备份流程

先用测试项目验证。安装 alpha 版本，在项目目录中运行 setup，再创建私有恢复点。

npm install -g aicodebackup@alpha
安装 alpha

为什么不要默认公开

AI 编程早期阶段可能包含未审查的提示词产物、临时文件、实验依赖和敏感业务上下文。公开仓库适合准备好的项目，不适合所有中间态。

私有备份适合哪些场景

  1. 还在验证产品方向的原型。
  2. 包含私有业务逻辑的工具。
  3. 尚未清理 README、许可证或安全边界的项目。
  4. 需要先稳定再决定开源的代码库。

仍然要检查本地文件

私有仓库不是忽略安全卫生的理由。备份前仍应避免提交密钥、令牌、cookie、恢复码和真实用户数据。

FAQ

我需要先学会 Git 吗？

不需要。AICodeBackup 仍然使用 Git 和 GitHub，但日常流程是 setup、backup、watch、doctor 和 restore。

备份会让仓库公开吗？

不会。为项目创建新仓库时，setup 默认创建私有 GitHub 仓库。

这能替代专业 Git 工作流吗？

不能。它是 AI 编程用户的安全层，用来在完整 Git 习惯建立之前先获得可靠的私有恢复点。
