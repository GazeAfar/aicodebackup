AI 编程安全层。

AI 编程安全层不是一个单点功能，而是一组固定保护动作：备份、检查、监控和安全恢复。

快速答案

AICodeBackup 的安全层覆盖项目初始化、备份、健康检查、风险监控和恢复。它的目标是让用户在高速 AI 编程时仍然知道代码是否被保护。

这层保护默认使用用户自己的私有 GitHub 仓库，不把源码托管到 AICodeBackup 的服务器。

试用备份流程

先用测试项目验证。安装 alpha 版本，在项目目录中运行 setup，再创建私有恢复点。

npm install -g aicodebackup@alpha
安装 alpha

安全层包含什么

  1. setup：建立 Git、GitHub 和私有远端仓库。
  2. backup：保存当前项目恢复点。
  3. doctor：检查认证、远端和本地状态。
  4. watch：监控变化规模和备份年龄。
  5. restore：恢复到新目录，避免覆盖当前工作。

它解决的核心问题

AI 让代码变化更快，但本地文件仍然脆弱。安全层把一次次临时改动变成可确认、可检查、可恢复的私有记录。

它不替代什么

它不替代代码审查、测试、密钥管理、正式发布流程或团队协作规范。它先解决个人 AI 编程阶段最常见的丢代码问题。

FAQ

我需要先学会 Git 吗？

不需要。AICodeBackup 仍然使用 Git 和 GitHub，但日常流程是 setup、backup、watch、doctor 和 restore。

备份会让仓库公开吗？

不会。为项目创建新仓库时，setup 默认创建私有 GitHub 仓库。

这能替代专业 Git 工作流吗？

不能。它是 AI 编程用户的安全层，用来在完整 Git 习惯建立之前先获得可靠的私有恢复点。
