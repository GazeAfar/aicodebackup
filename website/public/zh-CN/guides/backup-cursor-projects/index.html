自动备份 Cursor 项目。

Cursor 让本地项目里的 AI 改动非常快。备份层要跟上这个速度，避免只有编辑器里的本地文件可用。

快速答案

Cursor 项目适合使用私有 GitHub 仓库作为远端恢复点。AICodeBackup 负责 setup、backup、watch、doctor 和 restore，让备份动作保持简单。

每当 Cursor 完成一个可运行功能、准备大范围修改或结束会话时，都应该创建一次恢复点。

试用备份流程

先用测试项目验证。安装 alpha 版本，在项目目录中运行 setup，再创建私有恢复点。

npm install -g aicodebackup@alpha
安装 alpha

Cursor 项目的备份风险

AI 可以同时修改很多文件，用户可能还没来得及判断哪些变更最重要。如果没有远端恢复点，误删、坏重构或目录损坏都会扩大损失。

推荐使用方式

  1. 在 Cursor 项目根目录运行 setup。
  2. 确认 GitHub 私有仓库创建成功。
  3. 每个可运行阶段运行 backup。
  4. 跨文件改动前先备份。
  5. 用 doctor 检查最新工作是否已经推送。

适合配合 Cursor 的习惯

把 AI 提示词分成更小阶段，并在每个阶段跑通后备份。这样即使下一轮提示词效果不好，也能回到上一个稳定版本。

FAQ

我需要先学会 Git 吗？

不需要。AICodeBackup 仍然使用 Git 和 GitHub，但日常流程是 setup、backup、watch、doctor 和 restore。

备份会让仓库公开吗？

不会。为项目创建新仓库时，setup 默认创建私有 GitHub 仓库。

这能替代专业 Git 工作流吗？

不能。它是 AI 编程用户的安全层，用来在完整 Git 习惯建立之前先获得可靠的私有恢复点。
