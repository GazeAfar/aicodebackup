如何避免丢失 AI 生成的代码。

避免丢代码不是靠记忆，而是靠固定动作：有价值的阶段要有远端恢复点，危险改动前要有可回退版本。

快速答案

最小可行习惯是：开始前确认备份链路健康，关键节点运行 backup，长会话用 watch，恢复时永远恢复到新目录。

这套习惯能覆盖最常见的 AI 编程风险：误删、坏重构、会话中断、工具切换、依赖失败和本地目录混乱。

试用备份流程

先用测试项目验证。安装 alpha 版本，在项目目录中运行 setup，再创建私有恢复点。

npm install -g aicodebackup@alpha
安装 alpha

建立检查点习惯

把每个能运行的功能、每次大改前、每次结束会话前，都当成检查点。检查点不需要完美，但必须能让你回到一个已知状态。

推荐节奏

  1. 每天第一次打开项目时运行 doctor。
  2. 大提示词或跨文件改动前运行 backup。
  3. 功能跑通后立刻运行 backup。
  4. 长会话中开启 watch。
  5. 恢复时使用新目录，避免覆盖当前工作。

不要依赖临时复制

把文件夹复制成 project-final、project-final-2 只会让版本越来越难判断。私有 GitHub 恢复点更适合长期追踪变化和恢复。

FAQ

我需要先学会 Git 吗？

不需要。AICodeBackup 仍然使用 Git 和 GitHub，但日常流程是 setup、backup、watch、doctor 和 restore。

备份会让仓库公开吗？

不会。为项目创建新仓库时，setup 默认创建私有 GitHub 仓库。

这能替代专业 Git 工作流吗？

不能。它是 AI 编程用户的安全层，用来在完整 Git 习惯建立之前先获得可靠的私有恢复点。
