AI 编程的自动 GitHub 备份。

自动化的价值不是盲目提交所有东西，而是在风险升高时提醒你保存恢复点，并让备份状态可见。

快速答案

AICodeBackup 的 watch 模式可以监控文件变化、diff 大小和备份年龄。达到风险阈值时，你可以选择运行备份，或用 --auto 在合适场景自动备份。

对于 AI 编程，自动化应该配合人工判断：有意义的阶段保存，危险改动前保存，敏感文件先排除再保存。

试用备份流程

先用测试项目验证。安装 alpha 版本，在项目目录中运行 setup，再创建私有恢复点。

npm install -g aicodebackup@alpha
安装 alpha

自动化适合解决什么

它适合解决忘记备份、长会话无恢复点、项目变化过大却没有提醒的问题。它不适合替你判断哪些代码已经准备好发布。

推荐使用方式

  1. 先运行 setup 建立私有仓库。
  2. 正常节点手动运行 backup。
  3. 长时间 AI 会话时运行 watch。
  4. 确认阈值适合你的项目后再考虑 --auto。
  5. 定期运行 doctor 检查远端状态。

自动备份边界

自动备份不应该提交密钥、个人数据或不该进入仓库的构建产物。项目仍然需要 .gitignore 和基本安全检查。

FAQ

我需要先学会 Git 吗？

不需要。AICodeBackup 仍然使用 Git 和 GitHub，但日常流程是 setup、backup、watch、doctor 和 restore。

备份会让仓库公开吗？

不会。为项目创建新仓库时，setup 默认创建私有 GitHub 仓库。

这能替代专业 Git 工作流吗？

不能。它是 AI 编程用户的安全层，用来在完整 Git 习惯建立之前先获得可靠的私有恢复点。
