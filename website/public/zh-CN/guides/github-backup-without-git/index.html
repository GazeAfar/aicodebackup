不用学习 Git 也能做 GitHub 备份。

很多 AI 编程用户知道应该备份，但不想在第一个恢复点之前先学习一整套 Git 工作流。

快速答案

AICodeBackup 不替代 Git 和 GitHub，而是把最常用的安全路径包装成少量命令。你可以先建立私有备份习惯，再逐步学习更完整的 Git 工作流。

setup 会检查环境、连接 GitHub、创建或连接私有仓库；backup 会提交并推送当前项目；doctor 会告诉你当前工作是否已经被远端保护。

试用备份流程

先用测试项目验证。安装 alpha 版本，在项目目录中运行 setup，再创建私有恢复点。

npm install -g aicodebackup@alpha
安装 alpha

你不需要先记住的内容

第一次备份前，你不必先记住 init、remote、add、commit、push 的所有细节。AICodeBackup 仍然使用这些基础能力，但把它们放进更清晰的命令里。

推荐命令顺序

  1. npm install -g aicodebackup@alpha
  2. aicodebackup setup
  3. aicodebackup backup
  4. aicodebackup doctor
  5. aicodebackup watch
  6. aicodebackup restore --to ../safe-copy

什么时候再深入学习 Git

当你开始协作、需要分支、需要代码评审或要发布正式版本时，应该学习更完整的 Git 工作流。备份层解决的是第一阶段的丢代码风险。

FAQ

我需要先学会 Git 吗？

不需要。AICodeBackup 仍然使用 Git 和 GitHub，但日常流程是 setup、backup、watch、doctor 和 restore。

备份会让仓库公开吗？

不会。为项目创建新仓库时，setup 默认创建私有 GitHub 仓库。

这能替代专业 Git 工作流吗？

不能。它是 AI 编程用户的安全层，用来在完整 Git 习惯建立之前先获得可靠的私有恢复点。
